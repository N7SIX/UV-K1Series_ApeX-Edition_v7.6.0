# Version Update Summary - v7.6.5

Date: March-April 2026
Firmware build: v7.6.5 ApeX Edition
Focus: Spectrum analyzer UX, data integrity, and documentation alignment
Status: Implemented and validated

## Summary
v7.6.5 updates refined spectrum behavior and persistence consistency, including measurement-path hardening, visual/control adjustments, and synchronized user documentation.

## Key Changes
1. Spectrum measurement reliability
- Added per-sweep reset behavior to prevent stale scan-bin carryover.
- Improved RSSI handling safety with bounded read behavior and signed clamping.
- Strengthened persisted dB-range validation and fallback handling.
4. Airband (AM) restoration after Spectrum/Waterfall
- Fixed a bug where exiting Spectrum/Waterfall could break Airband (AM) reception until reboot. Now, all VFO state (modulation, AGC, compander) is fully restored, ensuring reliable Airband operation after using spectrum features.
2. Spectrum controls and persistence
- Peak-hold behavior and frequency-step limits refined for usability.
- Bandwidth/modulation behavior made more predictable in runtime paths.
- Runtime autosave and exit-save behavior aligned for consistent restore.

3. Documentation finalization
- Owner manual and spectrum guides updated for current behavior.
- Technical references and quick guides aligned with helper/model changes.

## Validation
- Build: ApeX build line validated during release cycle
- Diagnostics: documentation and edited source paths verified in workspace checks
- Notes: no intended compatibility break for supported hardware

## Impact
- Better scan trustworthiness and stable spectrum behavior across sessions.
- Improved user confidence through clearer and synchronized documentation.
