## Spectrum Data Integrity & RSSI Clamp Safety (April 4, 2026)

**SCOPE:** Spectrum sweep correctness, RSSI acquisition hardening, and still-mode calibration safety.

### Core Firmware Changes

1. **Per-Sweep Measurement Reset (Stale-Bin Prevention)**
   - Updated spectrum scan initialization to clear per-sweep RSSI measurement bins before a new sweep.
   - Prevents stale values from prior sweeps from appearing as retained peaks in fresh scans.
   - Reset best-peak display index to ensure redraw logic starts from clean sweep context.

2. **Still-Mode RSSI Calibration Safety**
   - Replaced direct unsigned RSSI offset arithmetic with signed intermediate math.
   - Added explicit clamp to valid RSSI bounds before conversion back to unsigned storage.
   - Prevents underflow/overflow wrap behavior when negative calibration offsets are used.

3. **RSSI Read Path Hardening + Persisted dBm Range Consistency**
   - Added bounded retry/timeout guard in `GetRssi()` for BK4819 busy-wait pre-read loop.
   - AM correction now uses signed intermediate arithmetic in `GetRssi()` with clamp to valid RSSI range.
   - EEPROM spectrum settings load now validates persisted `dbMin`/`dbMax` against the runtime-supported upper bound and restores safe defaults when `dbMin > dbMax`.
   - Prevents lockup-like polling behavior, signed wrap artifacts under AM fix, and confusing post-reboot display-scale resets.

### Affected Source

- `App/app/spectrum.c`
  - `InitScan()`
   - `GetRssi()`
   - `SPECTRUM_LoadSettings()`
  - `UpdateStill()`

## Spectrum Peak Hold, Frequency Step Limits & Decay Tuning (April 3, 2026)

**SCOPE:** New peak hold feature in spectrum scan mode, frequency shift step cap reduction, and dotted-line decay timing.

### Core Firmware Changes

1. **Peak Hold Feature (Scan Mode)**
   - Added `peakHoldRssi[128]` per-column maximum buffer.
   - `KEY_MENU` in scan mode toggles peak hold on/off (previously a no-op).
   - Active: dotted line (even columns) displayed at per-bin maximum above live trace.
   - Status bar shows `PH` label while peak hold is active.
   - Decaying: any bin without an active signal loses `PEAK_HOLD_DECAY_STEP` RSSI units per sweep (~5 second full fade at nominal scan rate).
   - Frequency change / `RelaunchScan()` clears the peak hold buffer so stale data never persists across a retune.

2. **Frequency Shift Step Limits Reduced**
   - Default (first-boot) step lowered from ±6000.00k to **±1000.00k**.
   - Hard runtime cap lowered from 6000.00k to **±2500.00k**.
   - EEPROM restore clamp updated to match (>250000 → fallback to 100000).
   - Prevents a single keypress from jumping most of the visible scan span.

3. **Decay Tuning**
   - `PEAK_HOLD_DECAY_STEP = 9` → approximately 5-second fade after signal goes inactive.
