Type guide:
- feat: new functionality
- fix: bug fix or safety correction
- refactor: internal restructuring without behavior change
- docs: documentation-only change
- chore: maintenance or tooling update

Writing standard:
- Keep subject line <= 72 characters.
- Use imperative mood ("add", "fix", "align").
- Keep message short and precise; avoid marketing language.
- Include only verifiable facts.
