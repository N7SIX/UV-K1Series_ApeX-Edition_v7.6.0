# Release Notes - N7SIX v7.6.5br2 Patch ApeX Edition

Release date: April 2026
Build target: UV-K1 Series / UV-K5 V3 (PY32F071 only)
Build identifier: n7six.ApeX-k1.v7.6.5br2
Edition: ApeX (all-in-one)

## Release Summary
v7.6.5br2-patch is a targeted spectrum analyzer quality-of-life and correctness release.
It resolves all known bugs in the STILL mode register inspector, fixes the BPF selector,
corrects an RSSI register overflow producing wildly incorrect dBm values, and adds a clear
user-feedback overlay when PTT is pressed with no active signal.

## Compatibility
- Supported: UV-K1 and UV-K5 V3 hardware with PY32F071 MCU
- Not supported: UV-K5 V1 or non-PY32F071 variants

## Key Updates

### 1. RSSI dBm reading root fix
- `RSSI_HW_MAX_VALUE` is now `511U` (BK4819 REG_67 bits [8:0] — physical hardware maximum).
- Previously `RSSI_MAX_VALUE = 65535U` was used as both a blacklist sentinel and an arithmetic
  clamp. Cast to `int16_t`, `65535U` evaluates to `−1`, causing every valid RSSI to exceed the
  comparison and be pinned to the sentinel. Result: STILL mode displayed dBm values in the
  range of 32,000+ instead of the real −130 to 0 dBm range.
- Fix: hardware ceiling `511` and sentinel `65535` are now separate constants.

### 2. STILL mode — RX audio ("helicopter noise") eliminated
- `EnterStillMode()` had no guard against entering RX without a valid peak — causing the
  BK4819 to be set up on frequency 0 Hz, producing loud oscillator noise ("helicopter").
- Guard added: check `peak.f == 0 || !IsPeakOverLevel()` before entering STILL RX.

### 3. PTT no-signal user feedback overlay
- Pressing `[PTT]` with no valid peak (or peak RSSI below trigger) previously silently returned.
- New behavior: a centered white-bordered overlay reading **NO DETECTED / ACTIVE SIGNAL** is
  shown for approximately 300 ms and automatically clears.
- Timer `noSignalMsgT` decrements in the main tick loop; `Render()` draws the box while active.

### 4. Register inspector — Arrow-down multi-field-change bug fixed
- Root cause: `LockAGC()` was called before reading `BK4819_REG_13`. Inside `LockAGC()`,
  `BK4819_InitAGC()` wrote `0x03BE` to REG_13, resetting the entire LNAs/LNA/VGA field triplet
  before the user's delta was applied.
- Fix: `BK4819_ReadRegister(s.num)` now captures the register value BEFORE `LockAGC()` is
  called. Only the target sub-field is modified within the captured value.

### 5. BPF selector — fixed (software-state index)
- Root cause: `GetBpfOptionIndex()` called `BK4819_ReadRegister(BK4819_REG_3D)`. REG_3D is
  effectively write-only — hardware reads return inconsistent values. Nearest-match resolution
  always landed on index 0 (8.46 kHz). The selector appeared stuck and could not be changed.
- Fix: `bpfSavedIdx` software variable is now the authoritative BPF index. `GetBpfOptionIndex()`
  returns `bpfSavedIdx` directly. `SetBpfOptionIndex()` writes hardware AND updates `bpfSavedIdx`.
- BPF selection is re-applied after every `ToggleRX()` and `RADIO_SetModulation()` call, which
  both previously clobbered REG_3D with the default value `0x2AAB`.

### 6. Rotary (wrap-around) navigation for all register fields
- All four register fields (LNAs, LNA, VGA, BPF) now use rotary navigation.
- Arrow UP at maximum → wraps to minimum (index 0).
- Arrow DOWN at minimum → wraps to maximum (top index).
- No more silent clamping at either end.

## Why It Matters
- STILL mode dBm values now match real-world signal strength.
- PTT entry is reliable without audio artifacts.
- Register inspector is fully functional for signal analysis and interference mitigation.
- BPF selection persists across all RX events.
