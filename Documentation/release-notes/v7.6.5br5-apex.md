# Release Notes - N7SIX v7.6.5br5 ApeX Edition

Release date: April 2026
Build target: UV-K1 Series / UV-K5 V3 (PY32F071 only)
Build identifier: n7six.ApeX-k1.v7.6.5br5
Edition: ApeX (all-in-one)

## Release Summary
v7.6.5br5 is a battery-readout accuracy and SysInf UX refinement release.
It hardens battery calibration handling with guarded conversion logic, unifies battery
voltage conversion paths, and moves battery diagnostics into SysInf for cleaner operation.

## Compatibility
- Supported: UV-K1 and UV-K5 V3 hardware with PY32F071 MCU
- Not supported: UV-K5 V1 or non-PY32F071 variants

## Key Updates
6. Airband (AM) restoration after Spectrum/Waterfall
	- Fixed a bug where exiting Spectrum/Waterfall could break Airband (AM) reception until reboot. Now, all VFO state (modulation, AGC, compander) is fully restored, ensuring reliable Airband operation after using spectrum features.
### 1. Battery calibration coherence hardening
- `MENU_BATCAL` now stores a coherent low/high calibration pair based on 5.2V and 7.6V references.
- Added range validation for stored low-point ADC and a safety fallback when low/high coherence is impossible.
- Prevents invalid calibration combinations that can produce unrealistic battery voltage output.

### 2. Guarded ADC to voltage conversion (shared path)
- Added `BATTERY_AdcToVoltage10mV()` in `App/helper/battery.c` with declaration in `App/helper/battery.h`.
- Uses two-point linear calibration only when calibration points are valid, monotonic, and physically coherent.
- Falls back to one-point high-reference scaling when only the high point is trustworthy.
- `BATTERY_GetReadings()` and spectrum status drawing now both use the shared conversion helper.

### 3. SysInf battery diagnostics integration
- Removed separate `BatInf` menu entry and integrated diagnostics into SysInf view.
- SysInf now shows voltage, SOC, health (`H`), and estimated remaining capacity (`C`) in compact 5x5 rendering.
- Added custom framebuffer pixel-shift helpers to align full-size author/version rows while preserving readability.
- Prevented duplicate SysInf text rendering by gating default line-loop output when custom rendering is active.

### 4. 5x5 glyph coverage extension for diagnostics strings
- Extended 5x5 glyph set to support additional characters needed by diagnostics output: `C`, `H`, `R`, `:`, `/`, `%`, and `.`.
- Added lowercase compatibility for common mixed-case battery strings.

### 5. Versioning and workspace tooling alignment
- Bumped firmware version strings and ApeX target name to `v7.6.5br5` in CMake and UI version definitions.
- Updated VS Code C/C++ workspace configuration for reliable embedded IntelliSense and compile-command mapping.

## Why It Matters
- Battery readouts remain stable and realistic even with imperfect field calibration input.
- Spectrum and main battery UI now use one conversion source, reducing drift between screens.
- SysInf provides actionable battery diagnostics without requiring a separate dedicated menu item.

## Migration/Upgrade Procedure

To ensure a clean and reliable upgrade to v7.6.5br5, follow this process:

1. **Restore Stock Firmware**
	- Use QPS-K Series Software to flash the official stock firmware.
2. **Factory Reset**
	- On the radio, select RESET ALL from the menu.
3. **Install ApeX Edition (Base)**
	- Flash N7SIX v7.6.5 ApeX Edition firmware. Do not configure settings yet.
4. **Apply ApeX Edition (Revision)**
	- Flash N7SIX v7.6.5br5 ApeX Edition update immediately after the base install.
5. **Restore User Data**
	- Use QPS-K Series Software to restore your saved channel backup and settings.

**Important:**
- For restoring stock firmware, always use the official QPS-K Series Software for flashing.
- For flashing N7SIX ApeX Edition (steps 3 and 4), use Multi-UVTools or UVTools for best compatibility and reliability.
- Do not skip the factory reset step.
- Do not configure the radio between steps 3 and 4.
- Use QPS-K Series Software for backup/restore of user data.
