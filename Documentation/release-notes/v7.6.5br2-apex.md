# Release Notes - N7SIX v7.6.5br2 ApeX Edition

Release date: April 2026
Build target: UV-K1 Series / UV-K5 V3 (PY32F071 only)
Build identifier: n7six.ApeX-k1.v7.6.5br2
Edition: ApeX (all-in-one)

## Release Summary
v7.6.5br2 is a safety and reliability-focused ApeX release that hardens low-voltage behavior, strengthens settings persistence, and improves spectrum operation consistency.

## Compatibility
- Supported: UV-K1 and UV-K5 V3 hardware with PY32F071 MCU
- Not supported: UV-K5 V1 or non-PY32F071 variants

## Key Updates

### 1. Low-voltage protection
- TX is blocked below calibrated 7.0V minimum safe threshold.
- EEPROM/flash persistence writes are blocked below calibrated 7.0V minimum safe threshold.
- Pending saves are deferred and retried once voltage is safe.

### 2. Persistence integrity
- Spectrum settings persistence uses native full-block flash writes for the 20-byte block.
- Added transactional settings snapshots (A/B):
  - versioned snapshot header,
  - payload checksum validation,
  - generation-based boot selection,
  - write-readback verification with retry/fallback.

### 3. Spectrum reliability and UX consistency
- Improved sweep-data integrity and RSSI handling safety.
- Immediate control behavior retained for modulation and bandwidth updates.
- More predictable settings restore behavior after reboot.

### 4. ApeX operational workflow alignment
- NOAA quick-access and long-press key behavior aligned for field workflow.
- Scan-range and spectrum key-path conflicts removed in current ApeX mapping.

## Why It Matters
- Reduces brownout-induced instability on weak or aging battery packs.
- Reduces probability of low-power/interrupted-write persistence corruption.
- Improves likelihood of recovering a consistent last-known-good configuration.
