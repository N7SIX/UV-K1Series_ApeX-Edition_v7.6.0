# Version Update Summary - v7.6.5br2 Patch

Date: April 2026
Firmware build: v7.6.5br2 ApeX Edition (patch)
Focus: Spectrum analyzer STILL mode correctness, register inspector usability, BPF selector
Status: Implemented and validated

## Summary
This patch resolves all known bugs in the spectrum analyzer STILL mode, including a critical
RSSI arithmetic overflow producing corrupt dBm values, a register-write race condition causing
multi-field changes on a single arrow-key press, a BPF filter selector that was permanently
stuck at the widest setting, and silent failure when PTT was pressed with no active signal.

## Key Changes

### 1. RSSI dBm fix — `RSSI_HW_MAX_VALUE = 511U`
- Separated the hardware RSSI ceiling (511, BK4819 REG_67 bits [8:0]) from the blacklist
  sentinel (65535).
- Eliminates `int16_t` overflow that produced 32000+ dBm display in STILL mode.

### 2. EnterStillMode() peak guard
- Guard added: `peak.f == 0 || !IsPeakOverLevel()` before calling `ToggleRX(true)`.
- Eliminates BK4819 initialization on a zero-frequency target, which produced oscillator noise
  ("helicopter audio") through the speaker.

### 3. PTT no-signal overlay
- `noSignalMsgT` counter shows a **NO DETECTED / ACTIVE SIGNAL** centered box for ~300 ms
  when PTT is pressed with no qualifying peak.
- Replaces silent return with explicit user feedback.

### 4. Arrow-key register inspector — read-before-lock fix
- `BK4819_ReadRegister(s.num)` is called BEFORE `LockAGC()` in `SetRegMenuValue()`.
- Prevents `BK4819_InitAGC()` → REG_13 reset from wiping LNAs/LNA/VGA during adjustment.

### 5. BPF selector — software-state index
- `bpfSavedIdx` static variable replaces hardware readback as the authoritative BPF index.
- `GetBpfOptionIndex()` returns `bpfSavedIdx`.
- `BPFRegOptions[bpfSavedIdx]` reapplied after `ToggleRX()` and `RADIO_SetModulation()`.

### 6. Rotary navigation for all register fields
- All four fields (LNAs, LNA, VGA, BPF) wrap at both ends.
- UP at max → wraps to 0; DOWN at 0 → wraps to mask/max-index.

## Validation
- Build: `./compile-with-docker.sh ApeX` — success (83/83 objects)
- Spectrum register inspector tested end-to-end: LNAs, LNA, VGA, BPF all functional
- PTT guard tested: overlay appears with no signal, suppressed when signal present

## Impact
- STILL mode is now a fully functional RF gain analysis and optimization tool.
- BPF selection is persistent and user-controllable within a spectrum session.
- dBm values displayed in STILL mode are accurate.
- No change to EEPROM schema, menu system, or other firmware modules.
