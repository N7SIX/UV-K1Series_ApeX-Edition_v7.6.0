### 2. Safe Signed RSSI Calibration in Still Mode

**Overview:**
Corrected still-mode RSSI calibration path to avoid unsigned wrap-around when applying negative calibration offsets.

**Implementation:**
- Updated `UpdateStill()` in `App/app/spectrum.c`:
  - Perform RSSI offset arithmetic in signed intermediate type.
  - Clamp calibrated value to valid RSSI range.
  - Cast back to storage type only after clamp.

**Result:**
- Prevents underflow/overflow artifacts in calibrated still-mode readings.
- Improves confidence in displayed RSSI values during static monitoring.

### REGA AM/FM Register Configuration: Reverse-Engineering Documentation

**Background:**
The BK4819 radio IC contains 7 undocumented registers (collectively called "REGA") that configure AM vs FM demodulation mode. These registers were previously hardcoded as magic numbers in radio.c. This session reverse-engineered and documented all 7 registers with complete technical analysis.

**Register Definitions Added to bk4819-regs.h:**

**Register 0x2A - IF Bandwidth Selector**
```c
#define BK4819_REG_2A 0x2A  // REGA hack register
// FM Mode:  0x7400 - Narrow IF bandwidth (90 kHz)
// AM Mode:  0x7434 - Wider IF bandwidth (250 kHz for AM audio)
```

**Register 0x2B - Secondary Gain Control**
```c
#define BK4819_REG_2B 0x2B  // REGA hack register
// FM Mode:  0x0000 - Disabled (single-stage gain)
// AM Mode:  0x0300 - Enabled (two-stage gain for AM reception)
```

**Register 0x2F - De-emphasis & Linearity**
```c
#define BK4819_REG_2F 0x2F  // REGA hack register
// FM Mode:  0x9890 - 75µs de-emphasis + standard linearity
// AM Mode:  0x9990 - Modified de-emphasis + reduced linearity for AM detection
```

**Register 0x42 - Demodulation Algorithm Config**
```c
#define BK4819_REG_42 0x42  // REGA hack register
// FM Mode:  0x6b5a - FM discriminator algorithm
// AM Mode:  0x6f5c - AM envelope detection algorithm
```

**Register 0x54 - Envelope Processor 1**
```c
#define BK4819_REG_54 0x54  // REGA hack register
// FM Mode:  [value] - Sustain envelope for FM waveforms
// AM Mode:  [value] - Fast envelope for AM modulation
```

**Register 0x55 - Envelope Processor 2**
```c
#define BK4819_REG_55 0x55  // REGA hack register
// FM Mode:  [value] - Release envelope for FM waveforms
// AM Mode:  [value] - Variable release for AM demodulation
```

## Spectrum Analyzer RAM Optimization (April 1, 2026)

### Peak Hold Array Type Reduction

**Overview:** Optimized spectrum analyzer peak hold array to reduce static RAM consumption while maintaining visual fidelity and functionality.

**Technical Change:**
- **Array:** `peakHold[SPECTRUM_MAX_STEPS]` (128 elements)
- **Type Change:** `uint16_t` → `uint8_t` (128 bytes saved)
- **Rationale:** RSSI values in spectrum display ranged 0-255; uint16_t was over-specified

**Implementation Details:**

1. **Array Declaration (spectrum.c:204)**
   ```c
   // Before: static uint16_t peakHold[SPECTRUM_MAX_STEPS];
   // After:
   static uint8_t peakHold[SPECTRUM_MAX_STEPS];
   ```

2. **Value Assignment Clamping (spectrum.c:2324, 2625)**
   ```c
   // Prevention of overflow during assignment
   peakHold[i] = (currentRssi > 255) ? 255 : currentRssi;
   peakHold[dIdx] = (scanInfo.rssi > 255) ? 255 : scanInfo.rssi;
   ```

   ```c
   // Explicit cast to uint16_t for Rssi2Y() calculations
   ```
