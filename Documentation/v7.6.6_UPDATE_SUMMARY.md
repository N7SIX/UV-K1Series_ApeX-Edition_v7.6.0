# Version Update Summary - v7.6.6

Date: April 2026
Firmware build: v7.6.6 ApeX Edition
Focus: Safety hardening, persistence integrity, and spectrum/runtime reliability
Status: Implemented and validated

## Summary
v7.6.6 consolidates multiple safety and reliability improvements for weak-power behavior, persistence recovery, and spectrum data integrity, while keeping ApeX workflow controls consistent for field use.

## Key Changes
1. Low-voltage safety controls
- TX blocked below calibrated 7.0V minimum threshold.
- EEPROM/flash writes blocked below calibrated 7.0V threshold.
- Pending saves deferred until safe voltage is restored.

2. Persistence integrity hardening
- Spectrum settings moved to native full-block flash persistence path.
- Added transactional A/B settings snapshots with checksum and generation-based recovery.
- Added write-readback verification with retry/fallback path.

3. Spectrum and workflow reliability
- Sweep/reset and RSSI arithmetic safety improvements.
- NOAA and key-path alignment updates for ApeX operation.
- Event-system integration groundwork and related architecture cleanup.

## Validation
- Build: ./compile-with-docker.sh ApeX (success)
- Diagnostics: edited files checked with no reported errors
- Notes: behavior aligned with updated manual and release documentation

## Impact
- Reduced corruption risk under weak battery or interrupted-write conditions.
- More predictable recovery to last valid settings state at boot.
