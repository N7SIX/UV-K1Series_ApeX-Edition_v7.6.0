# Release Summary - ApeX Edition v7.6.6

Date: May 2026  
Firmware: 🔺ApeX Edition v7.6.6  
Status: Stable Release

## 1. About This Firmware
🔺ApeX Edition is the all-in-one firmware for UV-K1 and UV-K5 V3. It combines spectrum tools, FM radio support, operational safety improvements, and menu/workflow enhancements in one unified build.

This release continues the v7.6.x direction: improving runtime reliability, user workflow consistency, and day-to-day usability while keeping compatibility with the ApeX feature set.

## 2. Getting Started
Verified guidance:
1. If you are already running 🔺ApeX Edition v7.6.x, flash/update normally and continue using the radio.
2. Hidden-menu access exists in firmware boot flow (PTT + SIDE BUTTON 1 path).

Legacy migration guidance (documentation-derived):
1. If you are coming from factory firmware or earlier than 🔺ApeX Edition v7.6.5, first flash 🔺ApeX Edition v7.6.5.
2. Enter hidden menu by holding PTT + SIDE BUTTON 1 while powering on.
3. Release keys and perform RESET ALL (menu 75/75).

Warning  
Due to memory-map changes across firmware generations, you may lose memory channels and user settings during migration.

## 3. Changelog

Notes:
- Verified items below are directly traceable in this repository.
- Legacy items are kept for historical context but are not currently traceable from repository evidence alone.

### Added
Verified:
1. Added transactional A/B settings snapshots with checksum and generation-based recovery.
2. Added write-readback verification with retry/fallback path for persistence writes.

Legacy (not currently traceable from repository evidence):
1. Added WIDE/NARROW filter selection for AM RX.
2. Added scan progress gauge behind ENABLE_FEAT_F4HWN_SCAN_PROGRESS.
3. Added homologated CTCSS index display in Rx/Tx CTCSS menus.
4. Added tool for decoding packed Quansheng stock firmware images.
5. Added low-voltage write deferral so pending settings saves wait until safe voltage is restored.

### Improved
Verified:
1. Improved spectrum settings persistence with snapshot-backed handling.
2. Improved SysInf naming/placement behavior in menu flow.

Legacy (not currently traceable from repository evidence):
1. Improved entering text into ChName in a multi-tap system (PR #310, thanks @mrkusypl).
2. Improved ChName editing with press EXIT backspace (Issue #281).
3. Improved Spectrum UI and workflow.
4. Improved FM Radio and UI (PR #268, thanks @mrkusypl).
5. Improved hidden menu behavior (starts from the first hidden item).
6. Improved NOAA and key-path alignment for ApeX workflow consistency.
7. Improved architecture baseline with event-system integration groundwork and cleanup.

### Fixed
Verified:
1. Fixed VFO step handling by synchronizing `StepFrequency` whenever `STEP_SETTING` changes (v7.6.6 commit path).
2. Fixed weak-power safety baseline by defining calibrated 7.0V minimum thresholds for TX and write safety constants.

Legacy (not currently traceable from repository evidence):
1. Fixed build when ENABLE_FMRADIO is disabled.
2. Fixed large-range spectrum stability (Issue #295).
3. Fixed weak-power safety behavior by blocking TX below calibrated 7.0V minimum threshold.
4. Fixed weak-power persistence risk by blocking EEPROM/flash writes below calibrated 7.0V threshold.
5. Fixed sweep/reset and RSSI arithmetic safety issues in spectrum/runtime paths.
