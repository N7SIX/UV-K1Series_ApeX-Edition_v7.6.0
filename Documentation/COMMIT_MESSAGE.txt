# Commit Message Standard (Short and Precise)

Use this file as the default commit-message template reference.

## Standard Template

<type>(<scope>): <short summary>

Why:
- <problem or reason>

What:
- <change 1>
- <change 2>

Impact:
- <result for firmware/users>

Validation:
- <build/test/diagnostics>

Files:
- <path 1>
- <path 2>

## Rules
- Subject line must be concise and clear.
- Prefer factual and measurable statements.
- Keep total message short; include only key technical points.
- Add safety/compatibility notes when relevant.
